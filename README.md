# OtpService

## Описание проекта

**OtpService** — серверное приложение для управления процессом подтверждения пользовательских операций с использованием одноразовых кодов (OTP).

Система обеспечивает регистрацию и аутентификацию пользователей, выдачу JWT-токенов, генерацию и проверку одноразовых кодов подтверждения, а также доставку уведомлений через различные каналы связи.

Приложение поддерживает разграничение прав доступа между пользователями и администраторами, хранение данных в PostgreSQL и ведение журнала работы системы.

Проект разработан как самостоятельный сервис аутентификации и подтверждения операций, который может быть интегрирован в корпоративные информационные системы, веб-приложения и мобильные сервисы.

## Основные возможности

* генерация одноразовых кодов подтверждения;
* проверка введённых пользователем OTP-кодов;
* отправка кодов по электронной почте;
* интеграция с SMS-шлюзами по протоколу SMPP;
* выдача и проверка JWT-токенов;
* хранение данных в PostgreSQL;
* логирование работы приложения;
* обработка ошибок и валидация входных данных.

## Используемые технологии

* Java 23
* Gradle
* PostgreSQL
* Jackson Databind
* JavaMail API
* OpenSMPP
* Apache HttpClient
* JWT (java-jwt)
* SLF4J + Logback

## Архитектура проекта

Проект реализован с использованием многослойной архитектуры, обеспечивающей разделение ответственности между компонентами системы.

### API Layer

Слой обработки HTTP-запросов и формирования ответов клиенту.

Основные компоненты:

* `AuthController` — аутентификация пользователей;
* `AdminController` — административные операции;
* `UserOperationController` — выполнение пользовательских операций;
* `AuthFilter` — проверка авторизации запросов;
* `RoutesDispatcher` — маршрутизация запросов внутри приложения.

### Service Layer

Содержит бизнес-логику приложения.

Основные сервисы:

* `UserService` — управление пользователями;
* `OtpService` — генерация и проверка OTP-кодов;
* `AdminService` — административные функции;
* `UserOperationService` — обработка пользовательских операций;
* `CheckExpirationScheduler` — контроль срока действия кодов подтверждения.

### Notification Layer

Отвечает за доставку уведомлений пользователям через различные каналы связи.

Поддерживаемые способы доставки:

* электронная почта;
* SMS-сообщения;
* Telegram;
* файловые уведомления.

Для выбора канала используется фабрика сервисов уведомлений (`NotificationServiceFactory`).

### Data Access Layer (DAO)

Обеспечивает взаимодействие с базой данных PostgreSQL.

Основные сущности доступа к данным:

* пользователи;
* OTP-коды;
* настройки OTP;
* пользовательские операции.

Для каждой сущности предусмотрен отдельный DAO-интерфейс и его реализация.

### Model Layer

Содержит основные сущности предметной области:

* пользователь;
* операция;
* OTP-код;
* конфигурация OTP;
* статусы и роли пользователей.

### Utility Layer

Набор вспомогательных компонентов:

* работа с JWT-токенами;
* хеширование паролей;
* обработка JSON;
* выполнение HTTP-запросов.

### Configuration Layer

Компоненты конфигурации приложения и управления подключением к базе данных PostgreSQL.


## Сборка проекта

Для сборки приложения выполните:

```bash
./gradlew build
```

После завершения сборки исполняемый JAR-файл будет размещён в каталоге:

```text
build/libs
```

Все необходимые зависимости автоматически копируются в каталог:

```text
build/lib
```

## Запуск приложения

Linux / macOS:

```bash
java -jar build/libs/OtpService-1.0-SNAPSHOT.jar
```

или

```bash
./gradlew run
```

Windows:

```bash
gradlew.bat run
```

## Конфигурация

Перед запуском приложения необходимо настроить параметры подключения к базе данных PostgreSQL, почтовому серверу и SMS-шлюзу в конфигурационных файлах проекта.

Основные параметры конфигурации:

* настройки подключения к PostgreSQL;
* SMTP-параметры для отправки электронной почты;
* параметры SMPP-соединения;
* секретный ключ для формирования JWT-токенов;
* настройки логирования.

В ходе работы были изучены:

* механизмы двухфакторной аутентификации;
* работа с JWT-токенами;

* интеграция с внешними сервисами доставки сообщений;
* организация логирования и обработки ошибок;
* сборка и сопровождение проектов с использованием Gradle.

Результатом работы является сервис подтверждения операций с использованием одноразовых кодов, который может применяться в корпоративных и пользовательских информационных системах.
